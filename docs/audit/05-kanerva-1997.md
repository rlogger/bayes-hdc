## Summary (200 words)
Kanerva (1997) crystallises the *Spatter Code* — a fully distributed binary representation in which every concept, role, filler, and composite is a {0, 1} hypervector of fixed dimension D ≈ 10⁴. **Claim:** a single algebraic substrate built from XOR, majority sum, and Hamming distance suffices to encode sets, sequences, and role-filler bindings, with capacity that grows with D and graceful degradation under bit noise. **Algorithms / equations:** binding `a ⊕ b` is component-wise exclusive-OR (self-inverse); bundling is component-wise *majority rule* (each output bit equals the majority over the summed bits, ties broken at random or by a fixed tiebreaker HV); similarity is normalised Hamming similarity `1 − dist_Ham(a, b) / D`, with 0.5 the random baseline. **Terminology coined / used:** "spatter code", "fully distributed representation", "majority rule", "holistic". **Demonstrated tasks:** symbolic structures (records / pairs / sequences), set-membership via similarity, mapping/translation across domains by the bundle of bound role-filler pairs. **Successor citations:** survives essentially unchanged as the BSC model in Kanerva (2009, §"Spatter code"), Kleyko et al. (2023, §2.3.6); is the bipolar twin of the binary MAP variant (Gayler 1998); the special-case-of-FHRR observation is due to Plate (1997).

## Substantive findings (for user review)
1. **Tie-breaking under even n.** Kanerva (1997) is explicit that majority bundling under an even number of summands requires a tie-break — either by a fixed random "tiebreaker HV" included in the sum (the formulation Kleyko 2023 §2.3.6 attributes to this paper) or by a stochastic per-component coin flip. The library's `bundle_bsc` (`bayes_hdc/functional.py:52-55`) computes `counts > shape_size / 2.0`, which silently maps ties to `False`. For odd n this is correct; for even n it introduces a deterministic asymmetry (more zeros than ones at the bundled bit when half the inputs are 1). Tests exist (`tests/test_functional.py`) but I have not verified whether the even-n case is covered. Worth either (a) documenting the tie-break convention as deterministic-bias-toward-0, (b) adding an optional `key` argument for stochastic tie-break consistent with Kanerva 1997, or (c) following the Hannagan-style convention (Kleyko 2023 §2.3.6) of XOR-ing the first/last summand on ties. This is a *correctness-of-attribution* point, not a bug.
2. **No explicit Spatter-Code term in code.** The class is named `BSC` and the docstring expands as "Binary Spatter Codes", which is good, but neither file uses Kanerva's original phrase "spatter code" or "fully distributed representation" — the BSC acronym is in fact a *posthoc* invention (Kleyko 2023 footnote 10: "This acronym was not used in the original publications but we use it here to be consistent with the later literature"). The library's terminology is consistent with current convention; not a fix, just a note.
3. **Bipolar isomorphism.** `MAP` with bipolar `{−1, +1}` random HVs is mathematically isomorphic to BSC under the map `0 ↔ +1, 1 ↔ −1, XOR ↔ ×, majority ↔ sign(sum)`. The library treats them as independent models with separate code paths. This is the modern convention (Kleyko 2023 §2.3.7) and is fine — but a one-line note in `DESIGN.md` §1 ("BSC and bipolar-MAP are isomorphic via the {0,1} ↔ {+1,−1} map") would make the relationship explicit and would head off confused user questions.
4. **No documented capacity bound.** Kanerva (1997) gives a back-of-envelope capacity argument for how many role-filler pairs can be bundled before retrieval breaks. The library implements bundle/unbind/cleanup but does not surface a capacity-vs-D guideline anywhere user-visible. A short "rule of thumb" in `DESIGN.md` (e.g., "for D = 10 000, expect reliable cleanup with ≤ ~30 superposed pairs") would be useful and is well within the scope of "design notes". Out of scope for a citation audit but flagged.
